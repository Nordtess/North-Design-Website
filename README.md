## üîß Technical Deep Dive

### Form Validation System

The contact form features a comprehensive validation engine that runs on every input change:

```javascript
// Real-time validation with visual feedback
function validateName() {
  const name = $nameInput.value.trim();
  const nameRegex = /^[a-zA-Z√Ö√Ñ√ñ√•√§√∂\s'-]+$/;

  if (name === "") {
    $nameError.textContent = "Please enter your name";
    return false;
  } else if (!nameRegex.test(name)) {
    $nameError.textContent = "Name can only contain letters";
    return false;
  } else {
    $nameError.textContent = "";
    return true;
  }
}

// Prevent submission if any field fails validation
$form.addEventListener("submit", function (e) {
  const validations = [
    validateName(),
    validateNumber(),
    validateEmail(),
    validateMessage(),
  ];

  if (!validations.every((v) => v === true)) {
    e.preventDefault(); // Block the submit!
  }
});
```

**Key Features:**

- Input fields change color based on validation state (red = error, green = valid)
- Error messages appear below each field with specific feedback
- Submit button is only functional when all validations pass
- Swedish character support (√Ö, √Ñ, √ñ) for international usability

### Dynamic Data Fetching with Axios

Jamie's project showcase demonstrates asynchronous data loading and client-side filtering:

```javascript
let projects = [];

// Fetch JSON data using Axios
axios.get("/data/projects-david.json").then((response) => {
  projects = response.data;
  render(projects);
});

// Live search filtering
filterInput.addEventListener("input", () => {
  const searchText = filterInput.value.toLowerCase();
  const filtered = projects.filter((p) =>
    p.title.toLowerCase().includes(searchText),
  );
  render(filtered);
});

// A-Z sorting
sortAZ.addEventListener("click", () => {
  render([...projects].sort((a, b) => a.title.localeCompare(b.title)));
});
```

**Why This Matters:**

- Demonstrates asynchronous JavaScript (Promises)
- Array manipulation methods (`filter`, `sort`, `map`)
- DOM rendering patterns without a framework
- Immutable data handling (spreading arrays before sorting)

### CSS Grid Mastery

The entire layout system uses CSS Grid for responsive, maintainable layouts:

```css
/* Responsive grid that adapts to screen size */
.employee-area {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 2rem;
  padding: 2rem;
}

/* Mobile-first approach */
@media (max-width: 768px) {
  .employee-area {
    grid-template-columns: 1fr;
    gap: 1.5rem;
  }
}
```

**Grid Advantages:**

- No floats, no positioning hacks, no flexbox gymnastics
- Automatically responsive with `auto-fit` and `minmax()`
- Clean, declarative layout code
- Consistent spacing with `gap` property

## üöÄ What I Learned

Building this project taught me:

- **Vanilla JavaScript is Underrated:** Frameworks are great, but understanding DOM manipulation, event delegation, and async patterns without abstractions is fundamental. You can build complex UIs without React.
- **CSS Grid vs. Flexbox:** Grid is superior for page layouts, Flexbox shines for component-level alignment. Using both appropriately makes responsive design trivial.

- **Form Validation Best Practices:** Client-side validation improves UX but should never replace server-side validation (which this project doesn't have, because satire doesn't need a backend).

- **Code Organization Matters:** Splitting CSS/JS by page while maintaining a shared template system makes collaboration easier. Group projects don't have to be messy.

- **Performance Without Frameworks:** Zero build tools, zero bundlers, zero dependencies (except Axios CDN). The entire site loads in milliseconds because there's no framework overhead.
